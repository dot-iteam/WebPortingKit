# Static and Embedded Resources

Serve filesystem files, bundle resources, and generated in-memory resources with MIME detection and cache validation.

## Static Files

Use ``staticFile(request:from:pathPrefix:mimeTypes:defaultMimeType:cacheControl:)`` directly inside a route, or register a prefix route with ``HTTPApplication/staticFiles(_:location:mimeTypes:defaultMimeType:cacheControl:)``.

```swift
app.staticFiles("assets", location: "/var/www/public")
```

A request for `/assets/css/site.css` maps to `/var/www/public/css/site.css`.

The static file helper protects the served root:

- Parent directory components are rejected.
- Encoded slash path components are rejected.
- Directories are not served.
- Symlink escapes outside the root are rejected.
- File metadata and bytes are read off the NIO event loop.

Rejected, missing, and non-file paths return `404 Not Found`. A file that exists but whose metadata or bytes cannot be read (for example, a permission error) returns `500 Internal Server Error`. The helpers register `GET` routes, so they do not by themselves answer `HEAD`.

## MIME Types

``HTTPMimeTypeRegistry`` maps file extensions to response content types. Extensions are normalized by trimming dots and lowercasing.

```swift
var mimeTypes = HTTPMimeTypeRegistry.default
mimeTypes.register("application/x-custom", for: ".custom")

app.staticFiles("assets", location: "/var/www/public", mimeTypes: mimeTypes)
```

Use `defaultMimeType` when unknown extensions need a custom fallback.

```swift
app.staticFiles(
    "downloads",
    location: "/srv/downloads",
    defaultMimeType: "application/octet-stream"
)
```

## Bundle Resources

Use ``bundleResource(request:in:subdirectory:pathPrefix:mimeTypes:defaultMimeType:cacheControl:)`` or ``HTTPApplication/bundleResources(_:bundle:subdirectory:mimeTypes:defaultMimeType:cacheControl:)`` when resources live in a bundle.

```swift
app.bundleResources("docs", bundle: .module, subdirectory: "Documentation")
```

When serving a directory tree from Swift Package Manager resources, prefer `.copy(...)` so the on-disk layout is preserved. `.process(...)` may flatten or rename resources.

## Embedded Resources

Use ``EmbeddedHTTPResource`` for small resources stored in code or generated in memory.

```swift
let robots = EmbeddedHTTPResource(
    id: "robots.txt",
    mimeType: "text/plain; charset=utf-8",
    bytes: Array("User-agent: *\nAllow: /\n".utf8)
)

app.get("robots.txt") { request in
    await embeddedResource(request: request, resource: robots)
}
```

For generated bytes, pass a data closure.

```swift
let css = EmbeddedHTTPResource(
    id: "theme.css",
    mimeType: "text/css; charset=utf-8",
    eTag: .generated(name: "length") { data in
        "\"theme-\(data.count)\""
    },
    data: { Data("body { color: black; }".utf8) }
)
```

## Metadata Stores

``EmbeddedResourceMetadataStore`` assigns a stable `Last-Modified` date to each resource ID and optionally caches generated ETags per resource ID and generator name.

Use the shared store for application-wide embedded assets. Use a custom store in tests or when distinct groups of resources need independent metadata lifetimes.

```swift
let store = EmbeddedResourceMetadataStore()

app.get("assets", "site.css") { request in
    await embeddedResource(request: request, resource: css, metadataStore: store)
}
```
